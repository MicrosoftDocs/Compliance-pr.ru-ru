## <a name="how-does-microsoft-365-centralize-and-report-on-audit-logs"></a><span data-ttu-id="97ade-120">Как Microsoft 365 централизует журналы аудита и сообщает о них?</span><span class="sxs-lookup"><span data-stu-id="97ade-120">How does Microsoft 365 centralize and report on audit logs?</span></span>

<span data-ttu-id="97ade-121">Многие типы данных журнала загружаются с серверов Microsoft 365 во внутреннюю вычислительную службу больших данных Cosmos.</span><span class="sxs-lookup"><span data-stu-id="97ade-121">Many different types of log data are uploaded from Microsoft 365 servers to an internal, big data computing service called Cosmos.</span></span> <span data-ttu-id="97ade-122">Каждая группа служб загружает журналы аудита с соответствующих серверов в базу данных Cosmos для агрегации и анализа.</span><span class="sxs-lookup"><span data-stu-id="97ade-122">Each service team uploads audit logs from their respective servers into the Cosmos database for aggregation and analysis.</span></span> <span data-ttu-id="97ade-123">Эта передача данных происходит через подключение TLS с проверкой FIPS 140-2 на утвержденных портах и протоколах с помощью проприетарного средства автоматизации, называемого загрузчиком данных Office (ODL).</span><span class="sxs-lookup"><span data-stu-id="97ade-123">This data transfer occurs over a FIPS 140-2-validated TLS connection on approved ports and protocols using a proprietary automation tool called the Office Data Loader (ODL).</span></span>
