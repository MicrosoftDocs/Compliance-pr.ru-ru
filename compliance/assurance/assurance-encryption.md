## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="706b0-117">Как Microsoft 365 шифрует данные при их передаче?</span><span class="sxs-lookup"><span data-stu-id="706b0-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="706b0-118">Продукты и службы Microsoft 365 используют протоколы транспорта, такие как TLS, для предотвращения перехвата несанкционированными сторонами данных клиентов во время их перемещения по сети.</span><span class="sxs-lookup"><span data-stu-id="706b0-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="706b0-119">К примерам данных при передаче относятся почтовые сообщения, которые находятся в процессе доставки, беседы, происходящие в собрании по сети, или файлы, реплицируемые между центрами обработки данных.</span><span class="sxs-lookup"><span data-stu-id="706b0-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="706b0-120">В Microsoft 365 данные считаются "в пути" всякий раз, когда устройство пользователя общается с сервером Майкрософт или сервер Майкрософт общается с другим сервером.</span><span class="sxs-lookup"><span data-stu-id="706b0-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="706b0-121">Exchange Online, SharePoint Online, Microsoft Teams и Office Online используют TLS, чтобы обеспечить конфиденциальность данных во время передачи.</span><span class="sxs-lookup"><span data-stu-id="706b0-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="706b0-122">Как Microsoft 365 управляет ключами, используемыми для шифрования?</span><span class="sxs-lookup"><span data-stu-id="706b0-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="706b0-127">Шифрование службы обеспечивает дополнительный уровень шифрования данных клиентов в Exchange Online, Microsoft Teams, SharePoint Online и OneDrive для бизнеса.</span><span class="sxs-lookup"><span data-stu-id="706b0-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="706b0-128">Шифрование службы предоставляет клиентам два варианта управления ключами шифрования: ключи под управлением Майкрософт или ключ клиента.</span><span class="sxs-lookup"><span data-stu-id="706b0-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="706b0-129">При использовании ключей под управлением Майкрософт службы Microsoft 365 автоматически создают и безопасно сохраняют корневые ключи, используемые для шифрования служб.</span><span class="sxs-lookup"><span data-stu-id="706b0-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="706b0-130">Клиенты с требованиями для управления собственными корневыми ключами шифрования могут использовать шифрование службы с ключом клиента.</span><span class="sxs-lookup"><span data-stu-id="706b0-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="706b0-131">С помощью ключа клиента клиенты могут создавать собственные криптографические ключи с помощью локального модуля аппаратного обслуживания (HSM) или Azure Key Vault (AKV).</span><span class="sxs-lookup"><span data-stu-id="706b0-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="706b0-132">Корневые ключи клиента хранятся в AKV, где их можно использовать в качестве корня одной из ключевых фигур, шифруя данных или файлов почтовых ящиков клиентов.</span><span class="sxs-lookup"><span data-stu-id="706b0-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="706b0-133">Код службы Microsoft 365 может получить доступ к корневым ключам клиента только опосредованно для шифрования данных и не может получить прямой доступ со стороны сотрудников Майкрософт.</span><span class="sxs-lookup"><span data-stu-id="706b0-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
