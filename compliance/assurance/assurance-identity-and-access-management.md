# <a name="identity-and-access-management-overview"></a>Обзор управления удостоверениями и доступом

## <a name="how-does-microsoft-365-protect-production-systems-from-unauthorized-or-malicious-access"></a>Как Microsoft 365 защищает производственные системы от несанкционированного или вредоносного доступа?

Microsoft 365 позволяет инженерам корпорации Майкрософт работать со службой без доступа к контенту клиентов. По умолчанию инженеры Microsoft 365 имеют нулевой постоянный доступ (ZSA) к контенту клиентов и не имеют привилегированного доступа к производственной среде. Microsoft 365 использует модель just-in-Time (JIT), just-Enough-Access (JEA) для предоставления инженерам группы обслуживания временного привилегированного доступа к рабочим средам, если такой доступ необходим для поддержки Microsoft 365. Модель доступа JIT заменяет традиционный постоянный административный доступ процессом, который инженеры запрашивают временное повышение привилегий при необходимости.

Инженеры, которые назначены группе служб для поддержки запроса на получение учетной записи группы обслуживания с помощью средства управления удостоверениями (IDM). Запрос на приемлемость инициирует серию проверок персонала, чтобы убедиться, что инженер выполнил все требования к проверке в облаке, прошел необходимое обучение и получил соответствующее утверждение управления перед созданием учетной записи. Только после того, как все требования к требованиям, можно создать учетную запись группы обслуживания для запрашиваемой среды.

Учетные записи группы обслуживания не предоставляет никаких постоянных прав администратора или доступа к контенту клиента. Когда инженеру требуется дополнительный доступ для поддержки службы Microsoft 365, он запрашивает временный повышенный доступ к ресурсам, требуемой им, с помощью средства управления доступом под названием Lockbox. Блокировка ограничивает повышенный доступ к минимальным привилегиям, ресурсам и времени, необходимым для выполнения назначенной задачи. Если авторизованный рецензент утверждает запрос на доступ к JIT, инженеру предоставляется временная учетная запись только с привилегиями, необходимыми для выполнения назначенной им работы. Эта временная учетная запись требует многофакторной проверки подлинности и автоматически удаляется по истечении утвержденного периода.

## <a name="how-does-microsoft-365-handle-remote-access-to-production-systems"></a>Как Microsoft 365 обрабатывает удаленный доступ к производственным системам?

Компоненты системы Microsoft 365 находятся в центрах обработки данных, географически отделенных от групп операций. У персонала центра обработки данных нет логического доступа к системам Microsoft 365. В результате сотрудники группы обслуживания Microsoft 365 управляют средой с помощью удаленного доступа. Сотрудникам группы обслуживания, которым требуется удаленный доступ для поддержки Microsoft 365, предоставляется удаленный доступ только после утверждения авторизованного руководителя. Весь удаленный доступ использует TLS, совместимый с FIPS 140-2, для безопасных удаленных подключений.

Microsoft 365 использует secure Admin Workstations (SAWs) для удаленного доступа группы обслуживания, чтобы защитить среды Microsoft 365 от компрометации. Эти рабочие станции специально разработаны для предотвращения преднамеренной или непреднамеральной потери рабочих данных, включая блокировку USB-портов и ограничение программного обеспечения, доступного в saw, тем, что требуется для поддержки среды. SaWs внимательно отслеживаются и отслеживаются для обнаружения и предотвращения вредоносного или непреднамеренного компрометации данных клиентов инженерами Майкрософт.

## <a name="how-does-customer-lockbox-add-additional-protection-for-customer-content"></a>Как защищенное содержимое добавляет дополнительную защиту для содержимого клиента?

Клиенты могут добавить дополнительный уровень управления доступом к своему содержимому, включив его. Если запрос на повышение прав на повышение прав в lockbox включает доступ к содержимому клиента, он должен утвердить его в качестве последнего шага в рабочий процесс утверждения. Это дает организациям возможность утверждать или отоать эти запросы, а также обеспечивает прямой доступ к клиенту. Если клиент отклоняет запрос на доступ к содержимого, доступ к запрашиваемому содержимому отклоняется. Если клиент не отклоняет или не утверждает запрос в течение определенного периода, срок действия запроса автоматически истекает без получения корпорацией Майкрософт доступа к контенту клиента. Если клиент утвердит запрос, временный доступ корпорации Майкрософт к его содержимому будет автоматически регистрироваться, проверяться и отзываться по истечении времени, назначенного для завершения операции устранения неполадок.
