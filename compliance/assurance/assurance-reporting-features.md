# <a name="microsoft-365-reporting-features"></a>Функции отчетов Microsoft 365

Функции отчетов в Microsoft 365 предоставляют различные отчеты аудита для Azure Active Directory (Azure AD), Exchange Online, управления устройствами, надзорной проверки и защиты от потери данных (DLP). Эти отчеты отличаются от отчетов об активности Microsoft 365.

## <a name="microsoft-365-reports-dashboard"></a>Панель мониторинга отчетов Microsoft 365

Панель мониторинга отчетов в предварительной версии Центра администрирования Microsoft 365 отображает действия по использованию в Microsoft 365. Глобальные администраторы Microsoft 365 или администраторы Exchange Online, SharePoint Online или Skype для бизнеса могут получить детализированное представление об использовании этой службы. Например, количество пользователей в определенной службе Microsoft 365, количество пользователей, активировавший приложения Microsoft 365 для предприятий (ранее называлось Office 365 профессиональныйplus), а также количество почты, которая проходит через организацию. Отчеты доступны за последние 7, 30, 90 и 180 дней.

## <a name="azure-ad-reports"></a>Отчеты Azure AD

Microsoft 365 использует Azure AD для проверки подлинности и управления удостоверением. Администраторы Microsoft 365 используют отчеты, созданные Azure, для выявления необычных действий и несанкционированного доступа к их данным. Вы можете использовать отчеты о доступе и использовании в Azure AD, чтобы получить доступ к целостности и безопасности каталогов для вашей организации. С помощью этих сведений можно выявлять и устранять возможные риски безопасности.

Отчеты Azure AD можно экспортировать в Microsoft Excel и соотнести с другими данными из Microsoft 365. Например, результаты поиска в журнале аудита могут дать представление о доступе, проверке подлинности и действиях на уровне приложений. Дополнительные отчеты об аномалиях и использовании ресурсов доступны в Azure AD Premium. Эти расширенные отчеты помогают повысить уровень безопасности и помогают реагировать на потенциальные угрозы путем применения аналитики по доступу к устройствам и использованию приложений. Дополнительные сведения см. в [отчете Azure Active Directory.](/azure/active-directory/reports-monitoring/overview-reports/)

## <a name="exchange-online-audit-reports"></a>Отчеты аудита Exchange Online

Отчеты аудита Exchange Online содержат сведения о доступе к почтовым ящикам и изменениях, внесенных администраторами в клиент Exchange Online. С помощью аудита почтовых ящиков вы можете использовать задачи, которые приводится в следующей таблице, для запуска отчетов и экспорта журналов аудита Exchange Online.

> [!NOTE]
> Необходимо включить ведение журнала аудита для каждого почтового ящика, чтобы события аудита для этого почтового ящика были сохранены в журнале аудита. Если ведение журнала аудита почтовых ящиков для почтового ящика не включено, события для этого почтового ящика не будут сохранены в журнале аудита и не будут отображаться в отчетах аудита почтовых ящиков. ## <a name="device-compliance-reports"></a>Отчеты о соответствии устройств требованиям

Вы управляете мобильными устройствами, подключенными к подписке, и обеспечиваете их безопасность с помощью Basic Mobility and Security для Microsoft 365. Мобильные устройства, используемые для доступа к электронной почте, календарю, контактам и документам, играют важную роль в том, чтобы сотрудники могли работать в любое время и из любого места. Очень важно защитить данные организации. Вы используете Basic Mobility and Security для Microsoft 365, чтобы настроить политики безопасности устройств и правила доступа. В случае потери или кражи вы также можете использовать Basic Mobility and Security для Microsoft 365 для стирки мобильных устройств.

В базовых отчетах о соответствии требованиям мобильности и безопасности представлен обзор политик, установленных организацией для защиты мобильных устройств, которые имеют доступ к данным Microsoft 365. Отчет позволяет фильтровать устройства по статусу соответствия требованиям, сообщениям о нарушениях, заблокированным устройствам, а также по количество устройств, стираемых в результате выполнения политик безопасности. Дополнительные сведения [см. в обзоре базовой мобильности и безопасности для Microsoft 365.](https://support.microsoft.com/office/overview-of-basic-mobility-and-security-for-microsoft-365-faa7d8e5-645d-4d59-839c-c8d4c1869e4a)

## <a name="data-loss-prevention"></a>Защита от потери данных

Политики DLP помогают управлять безопасностью и потоком информации в организации. Вы можете настроить политики для блокировки доступа к контенту, шифрования данных или уведомления пользователей о нарушениях политики с помощью подсказок политики DLP в приложении. Отчеты DLP предоставляют представление о количестве совпадений политик и правил, переопределее и ложных срабатываах.

Центр администрирования Microsoft 365 используется для просмотра сведений о количестве сообщений, обнаруженных политиками DLP. Отчеты DLP предоставляют сведения о совпадениях политик и правил для отправленной и полученной почты. Вы также можете просмотреть количество совпадений, переопределеев и ложных срабатывай для каждой политики за последние 24 часа с помощью Центра администрирования Exchange. При загрузке отчета Excel можно просмотреть дополнительные подробности, например, кто отправил сообщение, в какой день и сработали совпадения с политикой. Дополнительные сведения [см. в отчетах по обнаружению политик DLP.](/previous-versions/exchange-server/exchange-150/jj889415(v=exchg.150))

## <a name="auditing-in-yammer-enterprise"></a>Аудит в Yammer корпоративный

Yammer корпоративный предоставляет администраторам возможность экспортировать данные о действиях пользователей из сети Yammer с помощью API экспорта данных [Yammer](https://support.office.com/article/export-data-from-yammer-enterprise-b303d8f3-007d-4ad4-81f8-54fb1ecfb3f2)или вручную на странице администрирования сети Yammer. Возможность экспорта журналов ограничена сетевыми администраторами в Yammer. (Все глобальные администраторы Microsoft 365 являются администраторами сети Yammer.)

Экспортируемые данные:

| Filename | Описание |
|----------------------------|-------------------------------------------------------------------------|
| Users.csv | Все новые, ожидающих и приостановленных пользователей в сети |
| Messages.csv | Все сообщения в сети |
| Files.csv (метаданные) | Метаданные, такие как имя файла, URL-адрес API файла, ИД загрузщика, загруженные по адресу и т. д. |
| Files.csv (исходные файлы) | ZIP-файл исходных файлов, загруженных пользователями в Yammer |
| Topics.csv | Темы, созданные в сети |
| Pages.csv | Страницы (примечания), созданные пользователями в сети |
| Admins.csv | Все проверенные администраторы в сети |
| Networks.csv | Все внешние сети Yammer |

Данные Yammer корпоративный также доступны в отчетах об активности Microsoft 365. Кроме того, Yammer активно работает над дополнительным ведением журнала с помощью API действий управления Microsoft 365 и возможностью аргументировать данные с помощью Power BI. Дополнительные сведения об этих [функциях](https://fasttrack.microsoft.com/roadmap?filters=yammer) см. в план развития Office.
