# <a name="resiliency-and-continuity-overview"></a>Обзор устойчивости и непрерывности

## <a name="how-does-microsoft-ensure-business-continuity-in-the-case-of-a-disaster-or-other-threat-to-service-availability"></a>Как Майкрософт обеспечивает непрерывность бизнеса в случае аварии или другой угрозы доступности службы?

Группа управления непрерывной работой предприятия (EBCM) корпорации Майкрософт контролирует действия по управлению непрерывной работой и аварийному восстановлению в службах и облачных решениях Майкрософт. Представители бизнес-подразделений Майкрософт, например Microsoft 365, координируют работу с командой EBCM для разработки планов непрерывности бизнеса и проверки соответствия требованиям к непрерывности бизнеса.

В основе нашей методологии управления непрерывной бизнес-непрерывной деятельностью (BCM) является жизненный цикл BCM. Этот трехуготельный процесс разработан для адаптации, чтобы его можно было реализовать с помощью широкого спектра бизнес-моделей корпорации Майкрософт. Она начинается **с** этапа оценки для определения критически важных процессов и целей, которые должны быть включены в программу непрерывности бизнеса. На этапе оценки также требуется анализ влияния на бизнес(BIA). На **этапе** планирования основное внимание уделяется разработке и внедрению стратегий устойчивости и восстановления, а также документации их в официальных планах непрерывности бизнеса. Наконец, **проверка возможностей тестирует** планы непрерывности бизнеса и их реализации, чтобы проверить эффективность и выявить потенциальные улучшения.

Стратегия непрерывности бизнеса Microsoft 365 использует избыточность оборудования, сети и центра обработки данных. Репликация данных между центрами обработки данных обеспечивает высокую доступность и надежность в случае катастрофического инцидента. Кроме того, повышается устойчивость к mundane инцидентам, таким как изолированные сбои оборудования или повреждения данных.

## <a name="how-does-microsoft-test-business-continuity-and-disaster-recovery-plans"></a>Как Майкрософт тестировать планы непрерывности бизнеса и аварийного восстановления?

Политика microsoft Enterprise Business Continuity Management (EBCM) предусматривает, что все планы обеспечения непрерывности бизнеса и аварийного восстановления Майкрософт должны тестироваться, обновляться и пересматриваться на ежегодной основе. Службы Microsoft 365 проверяют свои планы непрерывности бизнеса не реже одного раза в год для политик EBCM. После создания и проверки отчетов о действиях для проверки, тестирования результатов и информирования об обновлениях плана в ответ на любые проблемы, обнаруженные во время тестирования.

Чтобы проверить стратегии устойчивости и восстановления с широким спектром потенциальных инцидентов, программа EBCM определяет несколько категорий тестовых сценариев, затрагивающих людей, местоположения и технологии. Уровень проверки, требуемой для каждой службы, зависит от ее критичности, при этом более важные службы получают более строгую проверку. Каждая группа служб Microsoft 365 тестирует свой план непрерывности бизнеса в соответствии с рекомендациями EBCM, чтобы оценить эффективность плана и готовность группы обслуживания выполнить этот план.

В рамках рекомендаций EBCM ежегодные проверки планов непрерывности бизнеса и проверки возможностей должны проходить в течение 12 месяцев после последнего отзыва. Проверка возможностей должна включать обзор вспомогательной документации, такой как BIA, для обеспечения ее точности. Корпорация Майкрософт делает результаты проверки возможностей для выбранных служб Microsoft 365 доступными для наших клиентов с помощью ежекварталов.

## <a name="how-does-microsoft-365-ensure-system-capacity-meets-demand"></a>Как Microsoft 365 обеспечивает способность системы отвечать требованиям?

Планирование мощности помогает группам обслуживания выделять ресурсы, необходимые для поддержки доступности служб Microsoft 365. Регулярное планирование емкости требуется в рамках программы EBCM корпорации Майкрософт. Группы обслуживания проверяют данные о емкости во время квартальных проверок, а также во время чрезвычайных ситуаций, которые требуют дополнительной проверки емкости.

Необработанные данные для планирования емкости поддерживаются каждой командой служб и включают такие показатели, как обработка системы, память и емкость оборудования. Запланированные проверки используют модель текущей емкости системы и проверяют ее с прогнозируемыми потребностями в чрезвычайных ситуациях. Если модель показывает пропускную способность, предлагаемые изменения емкости системы будут отправлены руководству группы обслуживания для рассмотрения. Утвержденные изменения включаются в новую модель перед внедрением инженерами группы обслуживания.

## <a name="how-does-microsoft-365-maintain-service-availability-during-routine-system-failures"></a>Как Microsoft 365 поддерживает доступность служб во время регулярных системных сбоев?

Microsoft 365 достигает устойчивости служб за счет избыточной архитектуры, репликации данных и автоматической проверки целостности. Резервная архитектура подразумевает развертывание нескольких экземпляров службы на географическом и физическом оборудовании, обеспечивая повышенную устойчивость к сбоям для служб Microsoft 365. Репликация данных гарантирует, что в разных зонах сбоя всегда существует несколько копий данных клиента, что позволяет восстанавливать критически важные данные клиента в случае их поврежденных, потерянных или даже случайно удаленных клиентом. Автоматическая проверка целостности повышает доступность данных путем автоматического восстановления данных, на которые влияет множество видов физического или логического повреждения.
