# <a name="supplier-management-overview"></a>Обзор управления поставщиками

## <a name="how-does-microsoft-manage-risk-related-to-suppliers"></a>Как Майкрософт управляет рисками, связанными с поставщиками?

Корпорация Майкрософт является партнером сторонних компаний для удовлетворения потребностей наших клиентов. Эти сторонние компании называются поставщиками или субобсказчиками. Безопасность и конфиденциальность поставщиков в корпорации Майкрософт регулируются нашей программой [SSPA ( SSPA)](https://www.microsoft.com/procurement/sspa?activetab=pivot1%3aprimaryr6)— набором корпоративных требований для всех поставщиков, которые сотрудничают с корпорацией Майкрософт для предоставления наших веб-служб. Хотя программа SSPA обеспечивает комплексное управление базой поставщиков, отдельные подразделения, такие как Microsoft 365, могут поддерживать дополнительные требования к своим поставщикам.

## <a name="how-does-microsofts-supplier-security-and-privacy-assurance-sspa-program-protect-customer-data"></a>Как программа SSPA (Microsoft Supplier Security and Privacy Assurance) защищает данные клиентов?

SSPA — это партнерство между службами закупок Майкрософт, корпоративными внешними и юридическими вопросами и корпоративной безопасностью, чтобы гарантировать соблюдение поставщиками принципов конфиденциальности и безопасности Корпорации Майкрософт. Область SSPA распространяется на всех поставщиков, которые обрабатывает персональные данные или конфиденциальные данные Майкрософт. Регистрация программы SSPA включает соблюдение требований Майкрософт к защите данных (DPR). DPR состоит из средств контроля безопасности и конфиденциальности, которые поставщики должны реализовать перед началом контрактной работы с Корпорацией Майкрософт. Все зарегистрированные поставщики ежегодно подтверждают соответствие требованиям DPR.

Требования DPR основаны на шести отдельных категориях обработки данных, для которые поставщик может быть утвержден в рамках регистрации в SSPA. Эти категории используются для определения риска, связанного со службами, которые поставщик предоставляет корпорации Майкрософт. Профиль обработки данных поставщика определяет, какие элементы управления DPR считаются соответствующими для обеспечения соответствующей защиты данных. Поставщики, которые обрабатывает данные, которые считаются более рискованными, должны соответствовать всем требованиям DPR, а также могут выполнять независимую проверку соответствия требованиям. Средства закупок Майкрософт проверяют состояние SSPA всех поставщиков, включая соответствие применимым частям DPR, прежде чем разрешать приобретение этого поставщика.

## <a name="what-types-of-subprocessors-provide-services-for-microsoft"></a>Какие типы субобъессоров предоставляют службы для Майкрософт?

"Субобработчик" — это третья сторона, которой корпорация Майкрософт занимается, чьи обязанности включают обработку персональных данных Майкрософт, для которых корпорация Майкрософт является обработчиком. Подпроцессоры Майкрософт подгруппы подгруппы по три отдельных категории. Каждый из них должен продемонстрировать соответствие SSPA, прежде чем обрабатывать данные клиента от имени Майкрософт.

-  Подпроцессоры технологий предоставляют технологии, используемые для предоставления определенных веб-служб Майкрософт. Если клиент развертывает одну из этих служб, подпроцессоры, указанные для этой службы, могут обрабатывать, хранить или иным образом получать доступ к данным клиента или персональным данным, помогая предоставлять эту службу.
- **Подпроцессоры** предоставляют службы, которые поддерживают, работают и поддерживают веб-службы. Если клиент развертывает одну из этих служб, указанные подпроцессоры могут обрабатывать, хранить или иным образом получать доступ к ограниченным данным клиента или персональным данным, предоставляя свои дополнительные услуги.
-  Подпроцессоры расширения персонала принимают две различные формы: в обоих сценариях персональные данные находятся только в помещениях Майкрософт, в системах Майкрософт и подчиняются политикам и надзору Корпорации Майкрософт.

    - Первая форма расширения персонала обеспечивает сотрудников, которые поддерживают, работают и поддерживают веб-службы Майкрософт. Выполняя свои обязанности, эти субобъессоры могут быть раскрыты для данных клиента или персональных данных. Например, подпроцессор может выполнять удаленные устранения неполадок на сервере Майкрософт и при этом может быть раскрыт для фрагментов данных клиента в журнале аварийного дампа сервера.
    - Вторая форма расширения персонала включает субобъессоры, которые работают вместе с штатными сотрудниками Майкрософт для поддержки, эксплуатации и обслуживания веб-служб Майкрософт. Эти субобъессоры могут быть выгружены псевдонимизированными данными в рамках работы наряду с сотрудниками Майкрософт, работающими без работы.

Технологии и подпроцессоры необходимы для реализации элементов управления доступом в соответствии с требованиями Майкрософт к защите данных (DPR). Эти требования соответствуют договорным обязательствам корпорации Майкрософт перед клиентами в условиях обслуживания Online (OST). Поставщики, которые выполняют работу по дополнению персонала, подвергаются таким же средствам контроля доступа для штатных сотрудников Майкрософт.

## <a name="how-does-microsoft-onboard-suppliers"></a>Как корпорация Майкрософт работает с поставщиками?

Сторонние поставщики должны подписать главным соглашением Майкрософт в рамках процесса входящей регистрации. Это соглашение регулирует отношения между корпорацией Майкрософт и ее поставщиками и обеспечивает согласованное управление отношениями с поставщиками. В рамках регистрации поставщики зарегистрировались в SSPA и должны выполнить все применимые требования, прежде чем их можно будет утвердить для любых категорий обработки данных. Подразделения Майкрософт могут создавать взаимодействия с поставщиками только в том случае, если действие обработки данных для этого взаимодействия соответствует категориям обработки данных, для которых утвержден поставщик.

## <a name="how-does-microsoft-notify-customers-of-changes-to-suppliers-who-process-their-data"></a>Как Корпорация Майкрософт уведомляет клиентов об изменениях поставщиков, которые обрабатывает их данные?

В соответствии с приложением Microsoft Online Service Data Protection (DPA) корпорация Майкрософт принимает дополнительные обязательства в отношении периодов уведомлений о добавлении любого подпроцессора. Периоды времени уведомления зависят от типа данных, которые подпроцессор будет обрабатывать от имени Майкрософт. Как указано в DPA, корпорация Майкрософт обязуется предоставлять клиентам уведомление по крайней мере за шесть месяцев до создания нового подпроцессора, который будет обрабатывать данные клиента. Корпорация Майкрософт предоставит уведомление о любых других персональных данных не менее чем за 30 дней. Уведомление предоставляется обновлением списка Microsoft Online Services [подпроцессора.](https://servicetrust.microsoft.com/ViewPage/TrustDocumentsV3?command=Download&downloadType=Document&downloadId=926b2cf5-6b6e-43ca-9bc3-f73e961aad5f&tab=7f51cb60-3d6c-11e9-b2af-7bb9f5d2d913&docTab=7f51cb60-3d6c-11e9-b2af-7bb9f5d2d913_Subprocessor_List)
